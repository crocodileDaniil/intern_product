# Credit Document Checks API

REST API для первичной проверки пакетов документов, планируемых к заявлению в банк для оплаты по льготным кредитам.

Сервис принимает пакет файлов, сохраняет его в локальное хранилище, фиксирует историю в PostgreSQL, определяет типы документов по именам файлов, проверяет комплектность для программ `federal` и `regional`, создает задачу нейромодуля-заглушки и предоставляет результат через API.

## Запуск

```bash
docker compose up --build
```

После запуска:

- API: `http://localhost:8000`
- Swagger UI: `http://localhost:8000/docs`
- OpenAPI JSON: `http://localhost:8000/openapi.json`

Контейнер приложения выполняет `alembic upgrade head`, затем запускает `uvicorn`.

## API

### Создать проверку

```bash
curl -X POST http://localhost:8000/api/checks \
  -H "Authorization: Bearer specialist-token" \
  -F "program=federal" \
  -F "files=@договор_47.pdf" \
  -F "files=@спецификация_47.pdf" \
  -F "files=@счет_47.pdf" \
  -F "files=@акт_47.pdf"
```

`program` принимает только:

- `federal`: обязательны договор, спецификация, счет, акт/УПД;
- `regional`: обязательны договор, счет, акт/УПД.

Ответ:

```json
{
  "check_id": "abc123",
  "task_id": "def456",
  "status": "check_in_progress",
  "status_label": "Пакет обрабатывается нейромодулем",
  "reason": "Базовая проверка пройдена. Файлы обрабатываются нейромодулем.",
  "issues": [],
  "documents": [
    {
      "name": "договор_47.pdf",
      "detected_type": "contract",
      "size_kb": 142
    }
  ],
  "extracted": {
    "message": "Базовая проверка файлов выполнена, пакет передан в нейромодуль. Извлечение реквизитов заменено заглушкой.",
    "task_id": "def456"
  },
  "checked_at": "2025-03-15T14:32:00Z"
}
```

Если пакет неполный, статус будет `rejected`, а в `issues` появятся ошибки уровня `error`. Неверный формат, размер больше 20 МБ и нераспознанное имя файла считаются предупреждениями уровня `warning`.

### Список проверок

```bash
curl http://localhost:8000/api/checks \
  -H "Authorization: Bearer user-token"
```

Возвращает `id`, дату, программу, статус и количество документов.

### Детальная проверка

```bash
curl http://localhost:8000/api/checks/{check_id} \
  -H "Authorization: Bearer user-token"
```

Возвращает полный результат проверки, список документов, issues и заглушку извлеченных данных.

## Технологии

- Python 3.11: версия из технических требований.
- FastAPI: REST API, OpenAPI-документация, валидация multipart/form-data.
- Pydantic: схемы запросов и ответов.
- SQLAlchemy: ORM-модели проверок и документов.
- PostgreSQL: хранение истории проверок, документов, версий и результатов.
- Alembic: миграции схемы БД.
- Docker Compose: запуск API и PostgreSQL одной командой.
- pytest: unit-тесты бизнес-логики.

## Архитектура

- `app/api/routes/checks.py`: HTTP-эндпоинты `/api/checks`.
- `app/services/checks.py`: определение типа документа, проверка комплектности, формирование статусов и сохранение проверки.
- `app/services/tasks.py`: `BackgroundTasks`-заглушка нейромодуля.
- `app/db/models/checks.py`: таблицы `checks` и `check_documents`.
- `app/schemas/check.py`: Pydantic-схемы API.
- `alembic/versions`: миграции БД.
- `storage/uploads`: локальное файловое хранилище внутри контейнера.

## Фоновые задачи

Для тестового задания используется `BackgroundTasks` FastAPI. После базовой проверки сервис создает `task_id`, возвращает статус `check_in_progress` и запускает заглушку нейромодуля. Заглушка не извлекает реальные реквизиты из документов, а сохраняет в `extracted` понятный ответ о том, что файл обработан тестовым нейромодулем.

Для промышленной версии этот слой лучше заменить на Celery + Redis/RabbitMQ, не меняя внешний API.

## Переменные окружения

Пример находится в `.env.example`.

- `DATABASE_URL`: строка подключения SQLAlchemy к PostgreSQL.
- `ADMIN_TOKEN`: токен администратора. Может создавать проверки и читать историю.
- `SPECIALIST_TOKEN`: токен специалиста казначейства.
- `USER_TOKEN`: токен пользователя с доступом на чтение.
- `UPLOAD_DIR`: директория для сохранения загруженных файлов.

## Тесты

Локально:

```bash
pip install -r requirements.txt
pytest
```

В контейнере:

```bash
docker compose run --rm app pytest
```

Тесты проверяют определение типа документа, предупреждения, ошибки комплектности и формирование итогового статуса.

## Компромиссы

- Ролевая модель сделана моковая. Потом нужно делать.
- Фоновые задачи потом нужно переводить на Celery, потому что `BackgroundTasks` FastAPI не рассчитан на тяжелую обработку документов под нагрузкой.
